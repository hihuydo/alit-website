Dual-Verify-Phase aktiv (Sprint BŌåÆC): `verifySessionDualRead(req)` in Edge-safe `src/lib/auth-cookie.ts` versucht primary-verify-first (mit userId regex-validate + tv-claim-validate, Legacy-JWT ohne tv ŌåÆ 0), f├żllt bei missing/invalid zur├╝ck. `setSessionCookie` schreibt primary UND cleart legacy atomar. `clearSessionCookies` cleart session+legacy+CSRF via `.set("", {maxAge:0, ...same-attrs})` (nicht `.delete()` ŌĆö `__Host-` prefix trap). **T1-S 3-gate `requireAuth`** (`src/lib/api-helpers.ts`): (1) JWT-verify via `verifySessionDualRead`, (2) env-scoped DB-tv-check via `getTokenVersion(userId, env)` aus `admin_session_version(user_id, env, token_version)` Table, (3) CSRF validate (`validateCsrfPair`) on POST/PATCH/PUT/DELETE.
